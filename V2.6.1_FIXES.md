# Version 2.6.1 - Bug Fixes & Enhancements

## Overview
Fixed 3 critical issues identified in v2.6.0 and added beautiful alert/tip box system for improved user experience.

---

## ✅ ISSUE 1: ENTER Key Not Transitioning to Gameplay - FIXED

### Problem
- Pressing ENTER in menu didn't properly transition to gameplay
- Users couldn't start games using keyboard

### Solution
- Enhanced ENTER handler in `handle_keydown()` for menu state
- Now shows beautiful alert box with game controls/tips before name entry
- Alert automatically closes on ENTER/SPACE/ESC and proceeds to name entry
- Callback system ensures smooth transition: menu → alert → name_entry → playing

### Code Changes
- Menu ENTER now triggers `show_alert()` with game tips
- Alert callback sets state to "name_entry" after closing
- Added global alert handling at top of `handle_keydown()`

---

## ✅ ISSUE 2: No Window Controls - FIXED

### Problem
- Players couldn't resize, minimize, close, or maximize game window
- Window was locked to 1280×800 fixed size
- No way to adjust screen layout

### Solution
- Added `pygame.RESIZABLE` flag to `pygame.display.set_mode()`
- Window now supports:
  - ✓ Resizing by dragging edges/corners
  - ✓ Minimize/Maximize/Close buttons (OS-provided)
  - ✓ Screen orientation changes tracked via VIDEORESIZE event
  - ✓ Full-screen support via OS controls

### Code Changes
```python
# Line ~117 in __init__()
self.screen = pygame.display.set_mode(SCREEN_RES, pygame.RESIZABLE)
```

- No way to show important tips or messages to players

## 📊 Implementation Details

### Window Resizing
- Resizing is fully supported by OS window manager
- Game maintains 1280×800 internal resolution
- Scaling handled by pygame automatically
- VIDEORESIZE events tracked for orientation changes

### Alert Box Rendering
- Alert drawn AFTER music badge but ON TOP of all screens
- Uses pygame.SRCALPHA for transparency
- Box width: 620px (fits in 1280 width)
- Box height: 380px + (tips count × 35px)
- Centered on screen

### Key Handling Flow (with Alerts)
```
handle_keydown(key)
  ├─ If alert_active:
  │  ├─ ENTER/SPACE/ESC → close_alert()
  │  └─ Return (don't process other keys)
  └─ Normal state-specific handling
```
